# Ruta laboral de 8 materias

Esta carpeta contiene una seleccion reducida del temario completo para entrar al mundo laboral en datos con el perfil:

- analista de datos fuerte,
- analista BI con Python y SQL,
- cientifico de datos junior orientado a datos tabulares.

No es la ruta mas completa ni la mas teorica. Es la ruta con mejor relacion entre empleabilidad inicial, tiempo de estudio y habilidades que aparecen con frecuencia en equipos reales.

## Veredicto critico sobre el temario completo

El temario completo de `ciencia_datos_python` si es suficiente como mapa universitario amplio y como base para una carrera seria en ciencia de datos. Tiene matematicas, estadistica, datos, machine learning, sistemas, MLOps, etica, producto e industria.

Pero no es suficiente por si solo para empleabilidad si se queda como lista de temas. Para que funcione laboralmente necesita:

1. Proyectos integradores desde temprano, no solo al final.
2. Evidencia publica: repositorios, notebooks limpios, dashboards y reportes ejecutivos.
3. Casos con datos reales o realistas de Mexico.
4. SQL y limpieza de datos con mucha practica, porque suelen pesar mas que modelos avanzados en puestos iniciales.
5. Comunicacion de resultados, no solo codigo.
6. Una herramienta BI o de dashboard, aunque no sea el centro del temario.
7. Git, terminal, ambientes y reproducibilidad como habito permanente.
8. Ingles tecnico suficiente para documentacion, entrevistas y lectura de errores.

La parte mas fuerte del temario completo es que no cae en el error de ensenar solo modelos. La parte debil es que tiene demasiadas materias para una ruta inicial: si el alumno intenta estudiarlas todas antes de aplicar a trabajos, puede tardar demasiado en construir portafolio.

## Que deje fuera y por que

- **Calculo, algebra lineal y probabilidad**: son muy importantes, pero para empleabilidad inicial se pueden estudiar en paralelo de forma aplicada. No las eliminaria de una carrera; solo no las pondria dentro de las primeras 8 si el objetivo es conseguir trabajo pronto.
- **Terminal, Git y Reproducibilidad**: deberia ser obligatorio, pero transversal. No lo cuento como materia porque debe practicarse en todas las entregas.
- **Arboles, ensambles y modelos no lineales**: entrarian justo despues de esta ruta. Son muy utiles para ciencia de datos tabular, pero primero conviene dominar regresion, clasificacion, validacion y comunicacion.
- **MLOps, cloud y data engineering avanzado**: son claves para crecer, pero suelen ser segunda etapa para analistas o cientificos junior.
- **Deep learning, IA generativa, grafos y bayesiano**: son potentes, pero no son el camino mas corto para el primer empleo generalista.

## Portafolio minimo para aplicar

Al terminar estas 8 materias, el alumno deberia tener al menos:

1. **Proyecto SQL + EDA**: datos publicos o simulados, consultas SQL, limpieza, visualizaciones y conclusiones.
2. **Proyecto de dashboard**: indicadores de negocio o sector publico, narrativa ejecutiva y archivo reproducible.
3. **Proyecto de regresion**: prediccion de una variable continua con diagnostico, metricas y limitaciones.
4. **Proyecto de clasificacion**: matriz de confusion, precision, recall, ROC/PR, ajuste de umbral y costo de errores.
5. **README profesional** por proyecto: problema, datos, metodo, resultados, como correrlo y limites.

## Tiempo recomendado

- Ruta intensiva: 6 a 9 meses.
- Ruta parcial con universidad o trabajo: 10 a 12 meses.
- Ritmo minimo util: 8 a 12 horas por semana con entregables quincenales.
